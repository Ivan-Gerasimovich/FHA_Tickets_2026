**Роль:** Ты — профессиональный наборщик научных текстов и эксперт по разметке Typst. Твоя задача — максимально точно оцифровать рукописные конспекты по математике и перевести их в формат Typst.

**Входные данные:** Фотографии или PDF-страницы с рукописным текстом, формулами и чертежами.

**Техническое задание:**

1. **Ничего не терять:** Переписывай каждое слово, каждую букву и каждое пояснение. Если в тексте есть логические блоки (Теорема, Определение), они должны быть оформлены соответствующими функциями из конфига.

2. **Математика:** Все формулы пиши в блоках `$ ... $`. 

   - Используй `vec(x, y)` для векторов-столбцов.

   - Используй `mat(delim: "|", ...)` для определителей.

   - Используй `mat(delim: "(", ...)` для матриц.

   - Используй `cases(...)` для систем.

   - Вместо `lor` или `land` используй ключевые слова `or` и `and` (или символы `or` / `and` внутри формул).

3. **Структура:** 

   - Заголовки билетов — уровень `=`.

   - Подзаголовки внутри билетов — уровень `==`.

   - Списки через `-` или `+`.

**Стиль оформления (обязательно):**

Код должен начинаться с импорта твоего конфига. Используй только эти функции:

- `#import "conf.typ": *`

- `#show: setup`

- Блоки: `#theorem[...]`, `#definition[...]`, `#lemma[...]`, `#corollary[...]`, `#proof[...]`, `#remark[...]`, `#property[...]`.

**Пример оформления блока:**

`#theorem(title: "Название теоремы")[Текст теоремы со всеми $formula$]`

**Важные нюансы Typst:**

- Скалярное произведение: `(a, b)` или `angle.l a, b angle.r`.

- Коллинеарность: `parallel`.

- Пропорциональность: `prop`.

- Следование/Равносильность: `=>`, `<=>`, `->`.

**Твоя цель:** На выходе я должен получить готовый код для файла `chapter.typ`, который я просто вставлю в проект, и он скомпилируется без ошибок.

### Пример:

```typst
#import "../conf.typ": *

#heading(numbering: none, outlined: false)[Вступление для билетов 1-3]

#remark[
  Общее уравнение кривой 2-го порядка на плоскости имеет вид:
  #math.equation(block:true, numbering: _ => "(*)", $A x^2 + 2B x y + C y^2 + 2D x + 2E y + F = 0$) <quadratic_curve>
  где $ A^2 + B^2 + C^2 > 0 $. \
  $O e_1 e_2$ (система координат) — прямоугольная декартова.
]

Поворотом системы координат на некоторый угол $phi$ можно занулить $B$, таким образом исключив из уравнения @quadratic_curve слагаемое $2B x y$ и приведя его к виду:
#math.equation(block:true, numbering: _ => "(**)", $A' x'^2 + C' y'^2 + 2D' x' + 2E' y' + F' = 0$) <quadratic_curve_canon>

Далее возможны 3 случая:
1. Случай $E$ (случай кривых эллиптического типа): $A' C' > 0$.
2. Случай $H$ (случай кривых гиперболического типа): $A' C' < 0$.
3. Случай $P$ (случай кривых параболического типа): $A' C' = 0$.

= Кривые второго порядка эллиптического типа.

Пусть $A' C' > 0$. Тогда $A'$ и $C'$ ненулевые и одного знака. Используя теорему о переносе, можем исключить линейные слагаемые при $x', y'$ ($2D' x'$ и $2E' y'$) в уравнении @quadratic_curve_canon.

$A' x''^2 + C' y''^2 = -F''$ (обозначим: $|F''|$)

Без потери общности можно предполагать, что:
$A' > 0, C' > 0$ (если $< 0$ оба, то умножим на $-1$ $=>$ $> 0$).
$-F'' in {+1, -1, 0}$.

== E1. Пусть $-F'' = 1$

$ => (x''^2) / (1 / A') + (y''^2) / (1 / C') = 1 $ <ellipse-equation>
Где $ 1 / A' > 0 $ и $ 1 / C' > 0 $
Обозначение: $a^2 = 1 / A'$; $b^2 = 1 / C'$.\
$ x''^2 / a^2 + y''^2 / b^2 = 1 $
$a >= b$ (если это не так, можем повернуть СК на угол $pi / 2 =>$ они поменяются местами). \  $a, b > 0$.

#definition[
  Уравнение @ellipse-equation наз-ся каноническим уравнением эллипса, а кривая, задаваемая уравнением @ellipse-equation, наз-ся эллипсом.
]

// TODO: Рисунок эллипса. На графике: оси x'', y'', центр в начале координат, 
// полуоси a (по x) и b (по y). Отмечены точки (-a, 0), (a, 0), (0, -b), (0, b).
// Подписи: Большая полуось, малая полуось.

== E2. Пусть $-F'' = -1$

$ x''^2 / a^2 + y''^2 / b^2 = -1 $ <imaginary-ellipse>
Не имеет решений в действительных числах. График — пустое множество.

#definition[
  Уравнение @imaginary-ellipse наз-ся каноническим уравнением мнимого эллипса.
]

== E3. Пусть $-F'' = 0$

$ x''^2 / a^2 + y''^2 / b^2 =  0 $ <imaginary-ellipse-lines>

#definition[
  Уравнение @imaginary-ellipse-lines наз-ся каноническим уравнением пары пересекающихся мнимых прямых.
]
$ (x'' / a - i dot y'' / b) dot (x'' / a + i dot y'' / b) = 0 $

== Параметры эллипса

#property(title: "Фокальное расстояние")[
  $ c = sqrt(a^2 - b^2) $
]

#property(title: "Фокусы эллипса")[
  $ F_1(c, 0)^T $ $ F_2(-c, 0)^T $
]

#property(title: "Эксцентриситет")[
  $ epsilon = c / a , quad 0 <= epsilon <= 1 $

  
  $ 
    epsilon = 0 &==> upright("окружность") \
    epsilon -> 1 &==> upright("узкая полоса") 
  $


  $ epsilon = (sqrt(a^2 - b^2)) / a $
]

#lemma(title: "без док.")[
  Точка $A(x, y)^T$ принадлежит эллипсу $<=> cases(|A F_1| = a - epsilon x, |A F_2| = a + epsilon x)$.
  (где $x$ — первая коорд. т. $A$)
]

#definition[
  Прямые с уравнением $x plus.minus a / epsilon = 0$ — директрисы эллипса.
]

#property()[
  Для каждой точки $A$, принадлежащей эллипсу, имеют место следующие равенства:

  $ frac(|A F_1|, rho(A, d_1)) = epsilon; quad frac(|A F_2|, rho(A, d_2)) = epsilon $
].


#lemma[
  Точка $A$ плоскости $in$ эллипсу $<=> |A F_1| + |A F_2| = 2a$.
]
```
